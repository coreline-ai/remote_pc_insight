# PRIVACY.md — Privacy & Data Handling

## 핵심 원칙
- 파일 **내용**(문서 본문/이미지 픽셀/오디오)은 수집하지 않습니다.
- 기본 업로드는 **요약(Level 0)** 이며 **경로/파일명은 숨김**입니다.
- 파일명/경로 전송은 사용자(또는 관리자)가 명시적으로 허용한 경우에만 가능합니다.

---

## 수집하는 것(예)
- 폴더 단위 용량/파일 수 (Downloads/Documents 등)
- 디스크 여유율(가능한 경우)
- 시작 프로그램 수(가능한 경우)
- 브라우저 캐시 후보 경로의 “용량 합”(경로 자체는 기본 숨김)
- 큰 파일 후보(기본은 경로 숨김)

---

## 수집하지 않는 것(예)
- 파일 내부 내용
- 브라우저 방문 기록의 내용
- 계정/비밀번호/개인식별정보
- 키 입력/화면 캡처

---

## 업로드 레벨 정책
- Level 0 (Default): 요약만 저장, 경로/파일명 숨김
- Level 1: 파일명만 저장(경로 제거)
- Level 2: 전체 경로 저장

권장: 서버에서 디바이스별 설정으로 강제하고, 업로드 전에 서버가 한 번 더 마스킹 적용.

---

## 사용자 제어권
- 디바이스 revoke: 분실/폐기 시 즉시 차단
- 데이터 삭제: 보존 기간/수동 삭제 정책 제공(v1 권장)
- 옵트인: 경로/파일명 업로드는 기본 OFF
